# 📊 TODO API 실측 벤치마크 결과 리포트

이 리포트는 Apple Silicon(M-시리즈) 환경의 Docker 컨테이너에서 각 프레임워크의 최신 안정화 버전을 대상으로 직접 측정된 데이터를 기반으로 작성되었습니다.

## 2. 심층 분석

### 🥇 리소스 가성비 및 처리량 1위: Go (Gin)
- **분석**: Go는 9.76MiB라는 **경이로운 메모리 점유율**과 가장 높은 RPS(167.89)를 보여주었습니다. 이미지 크기 또한 32.3MB로 다른 프레임워크들의 1/10 수준입니다.
- **특이사항**: 인프라 비용 절감이 최우선인 클라우드 네이티브 환경(K8s 등)에서 최고의 효율성을 자랑합니다.

### 🥈 안정적인 균형: Node.js (Express)
- **분석**: RPS 160.86으로 Go와 근소한 차이로 상위권에 속하며, 메모리 사용량(22.80MiB) 또한 매우 합리적입니다.
- **특이사항**: 성능과 리소스 면에서 여전히 매우 경쟁력 있는 "표준"임을 보여줍니다.

### 🥉 대규모 시스템의 기반: Java (Spring Boot)
- **분석**: 메모리 점유율이 154.3MiB로 가장 높지만, 이는 JVM의 특성입니다. RPS는 Python과 PHP보다 높게 측정되어 효율적인 요청 처리를 보여줍니다.
- **특이사항**: 리소스 소모는 크지만, 대규모 운영 환경에서 강력한 안정성을 제공합니다.

### 🐍 생산성의 강자: Python (FastAPI)
- **분석**: RPS는 142.79로 중간 수준이며, 이미지 크기(346MB)가 가장 큽니다.
- **특이사항**: 개발 생산성과 비동기 처리의 편의성에 강점이 있는 프레임워크입니다.

### 🐘 풍부한 기능의 대명사: PHP (Laravel 11)
- **분석**: 풀 프레임워크인 Laravel 11을 도입함에 따라 RPS는 118.37로 가장 낮고, CPU 점유율(6.40%)이 가장 높게 측정되었습니다.
- **특이사항**: 성능 수치는 낮지만, Laravel이 제공하는 방대한 생태계와 생산성, 보안 기능을 고려할 때 "트레이드 오프"가 명확한 선택지임을 보여줍니다.

## 3. 최종 결론

1. **가장 가볍고 효율적인 배포가 필요하다면?** 👉 **Go (Gin)**
2. **가장 대중적이고 성능 균형이 좋은 선택을 원한다면?** 👉 **Node.js (Express)**
3. **기업용 대규모 확장성과 안정성이 필요하다면?** 👉 **Java (Spring Boot)**
4. **AI/Data 연동 및 빠른 개발이 필요하다면?** 👉 **Python (FastAPI)**
5. **풀스택 기능과 생산성 위주로 개발한다면?** 👉 **PHP (Laravel)**

본 리포트는 벤치마크 툴(`benchmark-tool`)의 실측 데이터를 기반으로 작성되었습니다.
